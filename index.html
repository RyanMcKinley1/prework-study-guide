HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The p element represents a paragraph of block of text.
  • HTML ATTRUBUTES ARE ALWAYS DEFIND IN THE BEGINNING ELEMENTS AND NEVER IN THE CLOSING ELMENTS

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
  • External CSS are the most common way to style a webpage as id preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • DRY code: Do Not Repeat Yourself code, a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • When staging file we are telling git that we want to record the changes we made to the file. We do this by using “git add -A”
  • Commit changes by using Git commit -m “fixed bug in html……”
  • To push to github, Git push origin feature/starter-code Make sure to push to your branch and not the main branch!
